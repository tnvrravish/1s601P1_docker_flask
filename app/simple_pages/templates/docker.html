Docker

Docker is a containerization platform that is free and open source. It allows developers to package programs into containers, which are standardized executable components that combine application source code with the OS libraries and dependencies needed to run that code in any environment.

Docker technology conists of several terms like Containers, Docker images, Docker Hub etc.

A container is a standard software unit that encapsulates code and all of its dependencies so that the program can be moved from one computing environment to another fast and reliably. A Docker container image is a small, standalone software package that contains everything needed to run a program, including code, runtime, system tools, system libraries, and settings.

Docker Hub is the world's largest. library and community for container images. It can be used to store several docker images and even access prebuilt images. Helps us share our work with others. Images can be stored on repositories which are similar to real life folders on Docker Hub. We have used the following repository for this project : Docker Hub Repository

There are several commands in Docker and some of them are :-

  • docker build : This command is used to build a docker image.
  • docker run : This command is used to run the built image to create a container. there are several flags that can be used along with this command like -p which can take values like port numbers through which we can interact with our application.
  • docker ps : Used to get the list of all containers running and their status.
  • docker stop "the-container-id" : used to stop running of a specific container.
  • docker rm "the-container-id" : used to remove the container from the logs
  • docker push user_name/repository name : used to push the image to dockerhub repository.
  • docker network create : used to create a network in which multiple containers interact.
  • docker compose file : This file can be used as a blueprint for the execution of docker images and containers. Instead of runnning several dependencies one after other all dependencies can be put into compose file and with one command the desired working environment can be created.
  • docker compose up : This command is used to start up the compose file.
  • docker compose down : This command is used to terminate the compose file.

Steps to run the project :

  1. open cmd on your system.
  2. cd into the directory where the project is.
  3. use docker compose up --build -d command to run the image and create a container the -d flag allows us to run the container in detached mode
  4. now to access the terminal within the container use docker exec "container_id" /bin/bash command the allows us to use terminal in the container.
