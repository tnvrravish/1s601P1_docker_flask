Git is a distributed version control system that allows multiple people to work on the same project at the same time while also saving a history of all changes made. This will help to prevent code conflicts and give developers the opportunity to roll back files or entire projects to a previous version of their code.

Git can keep track of what modifications were made, who made them, when they were made, and why they were changed.

GitHub is a version control and collaboration platform for code. It allows you and others to collaborate on projects from any location.

Branches

Branching is a revolutionary feature in Version Control System. This allows us to diverge from the main line of development and continue to do work without messing with that main line. We can use git branch "Branch_name" option to create new branch which will have a copy of all the work until then on main line and from there we can continue with our work and do changes which will not affect the main branch code. git checkout command is used switch from one branch to other.

Merge

Git's approach of bringing a forked history back together is called merging. Thegit merge command allows you to combine the several development lines established by git branch into a single branch. To merge into the main branch we will have to commit our changes in the branch and the use the merge command. There can be few outcomes after this command, we can have successful merging and the new features from the branch are now available in the main branch to which the branch is merged. Or we can have something called as merge conflicts i.e. when there is inconsitency of data between two branches we get merege conflicts. These needs to be handeled and we can do that by overlooking what changes to be reflected and what not.

Commit

The git commit command takes a snapshot of the current state of the project's changes. Committed snapshots are "secure" versions of a project that Git will never modify unless you expressly tell it to. The git add command is used to promote or'stage' changes to the project that will be saved in a commit before running git commit. Git commit and git add are two of the most frequently used commands.

Tags

Git has the ability to mark critical points in the history of a repository. People usually use this feature to denote different versions of their code.

Repositories

In GIT, repositories are collections of files from various versions of a Project. These files are imported from the repository into the user's local server for additional updates and changes to the file's content. To create these versions and store them in a repository, a VCS (Version Control System) is utilized. Cloning is the process of copying content from an existing Git Repository with the aid of various Git Tools. The user receives the entire repository on his own PC after the cloning operation is completed. We can make several changes locally and then save it on the repositories using push command.
