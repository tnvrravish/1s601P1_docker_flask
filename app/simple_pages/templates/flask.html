Python-Flask

PyTest is a testing framework that lets users build test scripts in Python. It aids in the creation of simple and scalable API test scenarios. Several tests have been written in this project to test several instances of code and webpages. In this project within the tests folder, different python files are present to test different instances of the project. In simple_pages_test file I have written tests to check if the correct webpages are loading for certain responses and if 404 error status code is displayed for page not found.

There are several commands in Docker and some of them are :-

  • docker_flask <- root of the folder
    • docker-compose.yml : Docker orchestration file contains step by step process to run the container.
    • Dockerfile : Used to build a Docker Image.
  • workflows <- contains workflows blueprints
    • dev.yml : contains development workflow and is activated when an branch is merged into the main branch.
    • prod.yml : contains production workflow and is activated when data is pushed from local to github repository.
  • app folder <- contain webpages,controllers for servers, css, javascript files etc.
    • simple_pages<- contains controller for webpages of our for project.
      • templates<- contains all the webpages in our project, like index,flask,docker pages.
      • _init_.py : contains python function to render content from simple webpages combined with base html
    • static <- This folder contains all the static content for project like CSS, JavaScript and images.
    • templates<- contains template files like base.html that is common for most of the webpages.
    • run.py: used to create server for the API.
    • _init_.py : contains initiation of Flask application.
